# Structured prism meshing — design

**Date:** 2026-05-30
**Branch:** `feat/structured_discrete`
**Supersedes:** the planner + meshing design used on `feat/structured-clean4`,
summarized in [2026-05-30-structured-polygon-preprocessing.md](../../../2026-05-30-structured-polygon-preprocessing.md) and [2026-05-30-structured-meshing-pipeline.md](../../../2026-05-30-structured-meshing-pipeline.md).

## Goal

Extend `meshwell.cad_occ` so that a user can mark certain `PolyPrism`
entities as **structured** and get back, in the final mesh, those
volumes filled with **wedge** (linear prism) elements — generated by
extruding a 2D triangulation of the bot face in the z-direction —
while every other volume keeps its tet mesh and every shared interface
between structured and unstructured volumes is conformal.

The feature must support, in v1:

- Multi-layer structured slabs (`n_layers > 1`) for vertical refinement.
- Vertical stacks of structured polyprisms (cohorts), with internal
  z-plane interfaces shared cleanly between adjacent slabs.
- mesh_order-based **Policy B** carving when two structured polyprisms
  overlap in 3D (lower mesh_order wins, gets subtracted from higher).
- First-class arc preservation on lateral walls (curved boundaries
  produce true OCC arc edges, not polyline approximations, so the
  lateral structured mesh respects the underlying circle).
- **Imprint from unstructured neighbours above/below**: an unstructured
  PolyPrism whose bot/top face lands on a structured cohort's top/bot
  cuts the cohort face along the neighbour's outline. (Missing from
  the old design — was the source of conformality breaks at the cohort
  bot/top.)
- **Lateral imprint**: an unstructured neighbour whose z-range starts
  or ends strictly inside a structured cohort's z-extent splits the
  cohort vertically at that z-plane so the lateral wall can be
  imprinted cleanly. (Old design raised `StructuredMidHeightCutError`
  here.)

Not in v1: `recombine=True` (hex elements via gmsh recombine of
wedges), structured entities other than PolyPrism.

## Constraints (raise at planner stage)

The planner rejects the following inputs with explicit errors:

- A structured PolyPrism with `extrude=False` (z-buffered footprints).
  Only constant-XY-footprint prisms are structured-eligible.
- Two laterally-touching structured slabs (same cohort, same z-interval)
  with different `n_layers`.
- Any non-PolyPrism entity flagged structured.

## Why the old design was bloated

The old planner did the right shape of work but assembled the wrong
output. Concretely:

1. **One `TopoDS_Solid` per cohort**, with per-piece structure carried
   as Python metadata in `PhantomMap`. This forced an entire Phase 3
   "discrete cohort mesh" layer at mesh time (per-piece discrete 3D
   entities, interior interface stamping as discrete 2D, suppression
   of the empty envelope volume via `removeEntities`), and produced
   the multi-layer-cohort `getBoundary` corruption bug.
2. **Fixed-point face_partition iteration** for transitive cuts —
   needed because each slab's piece set depended on neighbours' piece
   sets, in turn depending on theirs. The iteration only existed
   because cuts were gathered per-slab, not per-cohort-per-z-interval.
3. **Arc data propagated through the planner** as Python (arc index,
   `PieceArcEdge`, `PieceLineEdge`, arc inheritance across z-steps,
   arrangement-edge arc fitting, `_coalesce_adjacent_arcs`,
   `_unify_concentric_arc_fits`). Arc handling was spread across
   six planner functions.
4. **Per-cohort planar arrangement with edge_ids** so Phase 3 could
   chain piece edges by corner connectivity into wires. Needed only
   because pieces were not first-class OCC volumes.
5. **Unstructured neighbours were not cut sources** for the cohort's
   piece decomposition, so the planner pieces did not align with what
   BOP actually produced after fragmenting cohort+unstructured →
   per-piece stamping had no clean mapping for the unstructured-cut
   sub-faces.

## Architecture

Five stages, each in its own module. Target total: ~700 lines (vs.
~3000 across the old `meshwell/structured/`).

```
meshwell/structured/
├── collect.py     — gather structured slabs from entity list
├── cohort.py      — Union-Find: structured slabs that touch (z or lateral)
├── decompose.py   — per cohort, per z-interval: sub-polygon decomposition
├── build.py       — assemble cohort TopoDS_Compound of N sub-solids
└── wedge.py       — pre_2d transfinite hints + pre_3d wedge stamping
```

### Stage 1 — `collect.py`

Walks the input entity list and partitions it into:

- `structured_slabs: list[StructuredSlab]` — one `StructuredSlab` per
  (PolyPrism with `structured=True`) × (z-interval from `buffers.keys()`).
  Carries `footprint`, `zlo`, `zhi`, `n_layers`, `mesh_order`,
  `mesh_bool`, `physical_name`, `source_index`, `identify_arcs`,
  `arc_tolerance`, `min_arc_points`.
- `unstructured_entities: list` — pass-through to `cad_occ`.

Errors on: `structured=True` with `extrude=False`; `structured=True` on
non-PolyPrism.

### Stage 2 — `cohort.py`

Union-Find over `structured_slabs`. Two slabs are merged into the
same cohort when:

- They share a z-plane AND their footprints intersect at that plane
  (face-touch), OR
- They share a z-interval AND their footprints intersect at that
  interval (lateral-touch).

Produces `cohorts: list[Cohort]` where each cohort holds its member
slabs and the union of their z-boundaries.

Lateral-touch with mismatched `n_layers` is detected here and raised.

### Stage 3 — `decompose.py`

For each cohort, compute the per-z-interval sub-polygon decomposition
in shapely. This is the only nontrivial planner stage.

**3a. Z-plane set.**

```
z_planes(cohort) =
    sorted set of (
        slab.zlo, slab.zhi  for slab in cohort.slabs
      ∪ {z in unstructured_entity.buffers.keys()
          for unstructured_entity in unstructured_entities
          if z lies strictly inside (cohort.zmin, cohort.zmax)
          and unstructured_entity z-overlaps cohort
          and unstructured_entity's XY footprint at z
              intersects cohort footprint at z}
    )
```

The unstructured contribution is what lets the planner pre-imprint
mid-height lateral cuts.

**3b. Per-z-interval footprint with Policy B carving.**

For each z-interval `(z_i, z_{i+1})`:

```
slabs_here = [s for s in cohort.slabs if s.zlo <= z_i and s.zhi >= z_{i+1}]
slabs_here.sort(key=lambda s: (s.mesh_order, s.source_index))

footprint = empty
for s in slabs_here:
    if s.mesh_bool:
        footprint = footprint ∪ (s.footprint − footprint)
    else:
        footprint = footprint − s.footprint
```

This is one shapely pass per z-interval; no iteration.

**3c. Pre-imprint cut sources.**

```
cuts(z_interval) =
    sibling structured slab boundaries in other cohorts at this z-interval
  ∪ unstructured neighbour XY outlines at z_i (cohort.bot of z-interval)
  ∪ unstructured neighbour XY outlines at z_{i+1} (cohort.top of z-interval)
  ∪ unstructured neighbour XY outlines at any z in (z_i, z_{i+1})
    intersected with the z-interval (lateral imprint)
```

For each z-interval: `polygonize(footprint.boundary ∪ cuts.boundary)`,
filter pieces whose representative_point lies inside `footprint`,
emit as `SubPiece(z_interval, sub_polygon, source_slabs)`.

**3d. n_layers propagation.**

Each `SubPiece` inherits `n_layers` from its source structured slab.
If multiple source slabs contributed (overlap zone), they all had to
agree per the Stage 2 invariant.

Output: `cohort.subpieces: list[SubPiece]`.

### Stage 4 — `build.py`

Construct one `TopoDS_Compound` per cohort containing N sub-solids
with shared internal TShapes. Bottom-up assembly:

1. **Unique vertex registry.** All `(x, y, z)` triples appearing in
   any subpiece, snapped to `point_tolerance`. Each yields one
   `TopoDS_Vertex` shared across faces that reference it.

2. **Unique edge registry.** All XY edges appearing in any subpiece
   boundary, at every z-plane they live on, and all vertical edges
   between adjacent z-planes at shared corners. Arc detection runs
   here: for each XY edge, run `decompose_vertices` on the polygon
   ring it belongs to and, if the vertex run fits a circle within
   `arc_tolerance`, build an arc edge via `GC_MakeArcOfCircle`;
   otherwise build a polyline edge. Each edge has one TShape, shared
   by every face that references it.

3. **Unique face registry.** Three kinds:

   - **Horizontal interior faces** at internal z-planes (z-planes
     strictly inside the cohort) where two z-adjacent subpieces share
     XY overlap → one shared face between them, built from their
     intersection polygon.
   - **Horizontal boundary faces** at the cohort's zmin and zmax.
     One face per subpiece at that boundary (no shared owner).
     **Pre-imprinted** by the decompose stage, so each is a single
     sub_polygon with neighbour outlines already baked into its edges.
   - **Lateral faces.** Per subpiece, one face per boundary edge of
     its sub_polygon, extruded between its z-interval. Two
     laterally-adjacent subpieces in the same z-interval share their
     interface face.

4. **Sub-solid assembly.** For each `SubPiece`, assemble a
   `TopoDS_Solid` from its six-ish faces (bot, top, laterals) via
   `BRep_Builder.MakeShell + MakeSolid`. All faces and edges point
   to the unique-registry TShapes.

5. **Compound assembly.** All sub-solids of one cohort go into one
   `TopoDS_Compound`. The compound is added to `cad_occ` as **one**
   argument, carrying its `mesh_order` (the minimum mesh_order across
   member slabs) and a sentinel `physical_name` that's later
   replaced per sub-solid.

6. **Metadata.** Build `slab_meta: dict[shape_key(sub_solid) ->
   SlabMeta]` where `SlabMeta = (slab_index, n_layers, physical_name,
   bot_face_key, top_face_key, lateral_face_keys)`. Survives BOP via
   the cad_occ piece-ownership pass (sub-solids that own themselves
   keep their key).

### Stage 5 — `wedge.py`

Two gmsh hooks, both narrow.

**`pre_2d_hook`** — transfinite hints on cohort lateral faces.

For each cohort sub-solid's lateral face (looked up via
`slab_meta[...].lateral_face_keys` → post-BOP gmsh tag via standard
`cad_occ` face mapping):

- Find the 1D vertical edges (endpoints differ in z).
  `setTransfiniteCurve(edge, n_layers + 1)`.
- `setTransfiniteSurface(face)` so the 2D mesh is a structured quad
  grid (no interior face nodes).

Lateral faces that gmsh would reject the hint on (multi-wire,
not-quadrilateral after BOP imprint) fall back silently to default
2D meshing — non-conformality on those faces will be caught by a
post-mesh validator.

**`pre_3d_hook`** — wedge stamping per cohort sub-solid.

For each cohort sub-solid (in `slab_meta`):

1. Look up the bot face gmsh tag and read its triangulation.
2. Look up the top face gmsh tag. For each bot node:
   - **Boundary node** (lies on a bot 1D edge): the corresponding
     top edge is the vertical-projection counterpart; the top
     boundary node already exists from 2D meshing. Match by
     tolerance nearest-XY.
   - **Interior node**: allocate a new top tag at `(bot.x, bot.y,
     zhi)`. Add to top face. If `n_layers > 1`, allocate
     intermediate-layer nodes at evenly-spaced z slices.
3. For each bot triangle: emit `n_layers` wedge elements stacked in z,
   into the sub-solid's gmsh 3D tag.
4. Register the slab's physical group on this 3D tag.

After the per-sub-solid loop, `gmsh.model.mesh.removeDuplicateNodes()`
dedups any boundary nodes that the per-sub-solid loop allocated
independently.

Then gmsh runs `generate(3)` with `Mesh.MeshOnlyEmpty=1`, which
tetrahedralizes only the unstructured volumes (cohort sub-solids
already have their wedge fill).

## Data contracts between stages

```python
@dataclass(frozen=True)
class StructuredSlab:
    source_index: int           # index in input entity list
    footprint: Polygon | MultiPolygon
    zlo: float
    zhi: float
    n_layers: int
    mesh_order: float | None
    mesh_bool: bool
    physical_name: tuple[str, ...]
    identify_arcs: bool
    arc_tolerance: float
    min_arc_points: int

@dataclass(frozen=True)
class Cohort:
    slabs: tuple[StructuredSlab, ...]
    z_planes: tuple[float, ...]   # all z-boundaries the cohort cares about

@dataclass(frozen=True)
class SubPiece:
    cohort_index: int
    z_interval: tuple[float, float]
    sub_polygon: Polygon
    source_slab_indices: tuple[int, ...]  # which StructuredSlabs contributed

@dataclass(frozen=True)
class SlabMeta:
    slab_index: int           # back to StructuredSlab.source_index
    n_layers: int
    physical_name: tuple[str, ...]
    bot_face_key: ShapeKey
    top_face_key: ShapeKey
    lateral_face_keys: tuple[ShapeKey, ...]
```

## Integration with `cad_occ`

`cad_occ.process_entities` currently does:

```
prepare_entities → sort by mesh_order → instantiate + cut → fragment_all
```

The structured pipeline runs as a **pre-pass** and a **post-pass**
around the existing cad_occ flow. No changes to the cut cascade or
`fragment_all` itself — they treat a cohort compound as one OCC shape
like any other.

**Pre-pass** (between `prepare_entities` and the cut cascade):

1. Run `collect → cohort → decompose → build` (Stages 1–4) on the
   structured subset of `entities_list`. Produces:
   - One `TopoDS_Compound` per cohort, each containing N
     `TopoDS_Solid` sub-pieces with shared internal TShapes.
   - `slab_meta: dict[ShapeKey -> SlabMeta]` keyed on each sub-solid's
     stable `_shape_key` (TShape pointer + orientation, already used
     by cad_occ).
2. Construct a synthetic `_CohortEntity` placeholder per cohort whose
   `instanciate_occ()` returns its compound and whose `mesh_order` is
   the minimum across member slabs. `mesh_bool` is True; the per-slab
   `mesh_bool=False` (void) carving has already been applied inside
   `decompose.py` Stage 3b.
3. Replace the structured slabs' positions in the entity list with
   one `_CohortEntity` per cohort (preserving the lowest source_index
   in the cohort, so insertion order ties resolve sensibly). Remove
   the slab entries; they're now folded into the cohort compound.

**cad_occ runs unchanged.** Each cohort enters as one entity → one
BOP argument. The cut cascade may carve the cohort against
unstructured neighbours with lower mesh_order; this is fine because
the carve preserves sub-solid TShapes (cuts only remove material from
the compound's outer boundary, not its internal faces).
`fragment_all` then runs the all-fragment BOP.

**Post-pass** (between `fragment_all` and XAO emission):

1. For each `OCCLabeledEntity` whose source is a `_CohortEntity`:
   walk its `shapes` list, look up each sub-solid by `ShapeKey` in
   `slab_meta`, and synthesize one fresh `OCCLabeledEntity` per
   sub-solid carrying that slab's `physical_name` and a synthetic
   `index` derived from the slab's `source_index`.
2. Replace the cohort entity in the result list with its expanded
   per-sub-solid entities.

`slab_meta` survives BOP because the planner-built sub-solid TShapes
are preserved through `BOPAlgo_Builder.Modified()`: a sub-solid that
isn't cut returns empty `Modified()` and we fall back to the original
shape (matches existing cad_occ behaviour at
[meshwell/cad_occ.py:351](../../../meshwell/cad_occ.py#L351)). A
sub-solid that IS cut (lower-mesh_order unstructured neighbour ate
part of it) returns its modified piece(s); the `ShapeKey` of the
modified piece is recorded into `slab_meta` alongside the original
during the post-pass so the wedge hook still finds it.

After XAO load, the wedge hooks
([Stage 5](#stage-5--wedgepy)) consult `slab_meta` keyed by
post-BOP `ShapeKey`. The bot/top/lateral face_keys are similarly
looked up via `builder.Modified()` and cached during the post-pass.

## Failure modes and validators

Planner-time (raise):

- Structured PolyPrism with `extrude=False`.
- Structured flag on non-PolyPrism.
- Lateral-touching structured slabs with mismatched `n_layers`.
- (Inherited from old) cohort footprint constancy is **not** enforced —
  z-stepped cohort outlines are now legal because sub-solids are
  per-z-interval first-class entities.

Build-time (raise):

- Sub-polygon assembly fails (degenerate polygon, OCC arc fit blows up).
- Sewn cohort compound has non-manifold edges (indicates a planner bug).

Mesh-time (warn, fall back to default mesher):

- Lateral face fails `setTransfiniteSurface` (multi-wire, > 4 boundary
  edges after imprint). Tet fallback for that one face only.

Post-mesh (validate):

- For every cohort sub-solid: bot triangle count × `n_layers` ==
  emitted wedge count. Otherwise raise.
- Wedge bot vertices align with bot face mesh node tags exactly.

## API additions

`PolyPrism.__init__` gains:

```python
structured: bool = False
n_layers: int = 1
```

No new entity class — `structured` is a per-instance flag, consistent
with `identify_arcs`. When `structured=True`:

- `extrude` must be True (raise otherwise).
- `n_layers >= 1`.
- `identify_arcs` defaults to True (arcs matter for lateral wall
  quad quality on curves); user can override.

## Out of scope for v1

- `recombine=True` (wedge → hex via gmsh recombine).
- Structured entities other than PolyPrism (e.g., a user-supplied
  triangle mesh extruded directly without OCC roundtripping).
- Z-buffered (taper/grow) structured prisms.
- Anisotropic refinement requests beyond `n_layers`.

## Open questions deferred to plan stage

- Exact shapely-to-OCC tolerance reconciliation (point_tolerance,
  cohort_envelope point snap, arc fit residual). Likely just inherit
  PolyPrism's existing point_tolerance / arc_tolerance defaults.
- Performance budget per stage. The 700-line target presumes
  `decompose.py` is ~250 lines, `build.py` is ~250 lines, others
  ~50–100 each.
